Description

An opaque data structure which represents a stack. It should only be accessed through the cadtstack_ functions.

struct stack_type StackADT
{
// No available fields
}
Note
To view the HTML rendered version of the C code for the implementation of this module, please visit: stack_adt.c.

Key Points

  • Relies on void pointers to allow manipulating elements of any type. See data_types.h.
  • Uses errno for managing stack underflows/overflows.
  • Dynamically allocated.
  • Stack object size can be fixed or variable.

Considerations

  • Clients are responsible for managing the memory space of the objects loaded to the structure.
  • No type safety.

StackADT* cadtstack_new ( size_t  size)

Creates a variable-size stack.

Allocates a stack with a minimum size of size that behaves as follows:

  • Doubles in size whenever full.
  • Halves if usage falls below 25% and the shrinking leaves it at a size at least equal to its original definition.

If the size argument passed is zero, errno is set to EINVAL. In case of failure to allocate memory errno is set to ENOMEM. For both cases the interpreted error message is outputted to stderr, and NULL is returned.

Parameters
sizeThe number of elements for initialization.
Returns
Returns a StackADT handle on success, NULL on failure.

◆ cadtstack_push()

Element cadtstack_push ( StackADT *  s,
Element  e 
)

Pushes an Element into a stack.

The behavior of the operation depends upon the type of stack object being pushed to:

  • If s is of fixed-size and there is no room to add e (Stack overflow), NULL is returned and errno is set to EPERM.
  • If s is of variable-size with no room for e and the system fails to allocate memory NULL is returned and errno is set to ENOMEM.
Parameters
sThe stack to push to.
eThe element to push into s.
Returns
Returns e on success, NULL on failure.

◆ cadtstack_pop()

Element cadtstack_pop ( StackADT *  s)

Pops an item from a stack.

Returns and removes the last element added to the stach. If s is empty (Stack underflow), NULL is returned and errno is set to EPERM.

  • If s is a variable-size stack and its usage is below 25%, the stack size is halved.
  • If the reallocation of memory fails, ENOMEM is set, but the top element is still popped from the stack.
Note
Client-side is responsible for deallocating the memory in-use by all elements in s.
Parameters
sThe stack to pop from.
Returns
Returns NULL on underflow, an Element otherwise.
